# day1-github-practice
In day-1 session , I learned about the difference between git and github, why we need it and the common commands like repo,clone,push,pull,commit,branch and creation of new repo with visibility(public/private).
GIT    : It is used for version control means whatever the changes that made in the code it will keep track of that changes.
GITHUB : It is cloud based platform where we can host our folders and also user can fetch  whatever they want in that folder.
DEFNITIONS OF COMMONLY USED COMMANDS IN GITHUB:
1. REPOSITORY : It is like a folder where all files resides inside it.
2. COMMIT     : If we want to upload some files or something to github after selecting files we use this command.
3. PUSH       : If we want to upload some files or something to github then we use this command.
4. PULL       : It is used to fetch the files that we want.
5. CLONE      : It is used to create a exact copy of existing one.
